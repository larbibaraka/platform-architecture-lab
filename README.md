# Platform Architecture Lab

Most engineers learn tools.
I'm rebuilding foundations.

This repository documents my 16-week journey into first principles of infrastructure systems.

## Objective

To deeply understand how infrastructure behaves under failure — not just how to use tools.

The focus:

- Linux internals
- Container behavior
- Kubernetes reconciliation
- Terraform state modeling
- System design thinking

Not to collect certifications.
But to build architectural depth.

## Structure

- phase-0-linux → Linux fundamentals & system behavior
- phase-1-docker → Container runtime internals
- phase-2-kubernetes → Orchestration & reconciliation
- phase-3-terraform → Infrastructure state modeling
- phase-4-architecture → Advanced concepts & capstone

Each phase contains:

- Labs
- Experiments
- Notes
- Failure analysis
- Reflections

This is a living lab.
